CH.01

LLM 的演進

5–10 min
  • LLM 發展脈絡：從早期語言模型(BERT)到 ChatGPT、Claude 等大型模型的躍進
  • 模型能力的快速演進：本質仍是"預測"，但理解、推理、生成能力發生質變
  • A Bitter Lesson 苦澀的教訓 By Rich Sutton：規模與通用計算終將勝過人工知識工程
參考資料
  • The Bitter Lesson 苦澀的教訓
  • 大語言模型簡史：從 Transformer（2017）到 DeepSeek-R1（2025）的進化之路
  • El Cerebro detrás de OpenAI y Google 🤖 | Łukasz Kaiser, Lead Researcher en OpenAI
  • Transformer 作者 Łukasz 揭示「推理典範」取代 LLM
CH.02

什麼是 Agent？

20 min
  • LLM 賦予機器高度自主性，以 LLM 為核心打造的服務完全顛覆傳統產品範式
  • Agent 的核心運作邏輯：Perception → Memory → Planning → Action (Plan-and-execute mode)
  • Simon's Scissors：模型表現由 Cognition 模型參數結構 與 Context 輸入脈絡 共同決定
  • 由此帶出兩個新興領域：Prompt Engineering → Context Engineering、Data → Environment
  • 補充案例：資料結構清晰、API 整理完善的組織導入 Agent 效率更高——資訊環境的品質決定 Agent 能力上限
參考資料
  • LLM Powered Autonomous Agent By Lillian Weng
  • 什麼是 Context Engineering 上下文工程？
  • 12-Factor Agents: Patterns of reliable LLM applications — Dex Horthy, HumanLayer
CH.03

數位服務本質的轉變

20 min
  • 服務結構演進：UI + Flow（使用者自行操作）→ Agent（使用者交付任務）
  • User 的擴展：使用者不再只是人，Agent 也是 User——介面、資料格式、回應結構都需要重新思考
  • 因應 Agent 新軟體架構，設計師任務產生轉變：從設計操作流程，轉向設計系統編排、任務邊界與行為準則；在User Journey Mapping 之上建構 Intent-System Mapping
  • 服務研究與測試的轉變：從觀察「人如何操作介面」到理解「Agent 在哪個環節認知失敗」；Observability（可觀測性）為構建Agent服務的基石，需追蹤模型決策過程
參考資料
  • Andrej Karpathy: Software Is Changing (Again)
  • How to Embrace the Great UX Paradigm Shift to Agentic Experience Design
CH.04

職能角色的重塑

5–10 min
  • 過去的職能分工源自舊結構需求，服務結構改變則各角色被期待交付的產出跟著轉變
  • 各職種的危與機：當 Agent 可以一鍵生成原型、規格、程式碼、分析行為數據，不可替代的核心價值是什麼？
  • 職能邊界模糊化的時代來臨：個人全能化 vs. 更高抽象層次的再分工
  • 稀缺的將是「能清楚定義問題、設定目標、判斷產出品質」的整合型判斷力—— How → WHY 轉移，Product Sense將成為核心競爭力
參考資料
  • The design process is dead. Here's what's replacing it. | Jenny Wen (head of design at Claude)
  • Agent Engineering: A New Discipline
  • AI 時代的產品判斷力：Product Sense 全指南 — Part 1：解構產品品味
實作 Part 動手體驗

實作 (1)　用 Agent 協助開發前後端網路應用
——無技術背景的起手式

30 min
目標：讓無技術背景的研究生建立起手式的心智模型，了解與 Agent 協作開發前需要具備哪些基礎概念。
  • 建立基礎心智模型：前後端的分工概念、程式開發的專案結構是什麼
  • 開發環境的基本認識：虛擬環境、套件管理（pip / npm）、常見錯誤排除
  • 資料儲存的類型選擇：關聯式 vs. 非關聯式資料庫的差異與使用情境
  • 如何與 Agent 協作開發：怎麼描述需求、怎麼拆解任務、怎麼看懂 Agent 產出的程式碼
  • 實作：從一句話需求出發，透過 Agent 逐步建立一個可運行的簡單網路應用

實作 (2)　搭建一個 Agent Prototype
——工具調用與 Context 加載

30 min
目標：讓學生快速體驗從零搭建一個具備工具調用能力的 Agent，理解 Context 加載的運作方式。
  • 設計一個具備工具調用（Tool Use）能力的 Agent：定義工具、設定觸發條件
  • Context 加載的實踐：如何將知識、文件、歷史對話注入 Agent 的運作流程
  • 快速實現一個可展示的 Agent Prototype，體驗從零到可用的完整流程
