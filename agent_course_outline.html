CH.01

LLM 的演進

5 min
  • LLM 的本質與演進脈絡：從 BERT 到 ChatGPT、Claude，模型能力發生質變——但本質仍是「預測下一個詞」，理解這個本質有助於後續評估 Agent 的能力邊界、構思 Agent harness
  • A Bitter Lesson：規模終將勝過人工知識工程（Rich Sutton）：通用計算能力的規模化，長期表現優於人工設計的知識工程——解釋了為何 LLM 能在短時間顛覆多個領域。AI 進步很快，可能今天相信的東西明天又改變了，保持開放態度、持續學習反思很重要
  • Software 3.0（Andrej Karpathy）：以自然語言提示 LLM：神經網路透過 LLM 變得可程式化，它是一種新型態的電腦。提示詞（prompts）成為可編程 LLM 的程式，而且這些提示是用自然語言寫成的

  • 大語言模型簡史：從 Transformer（2017）到 DeepSeek-R1（2025）的進化之路

CH.02

什麼是 Agent？

20 min
  • Agent 的運作邏輯：ReAct Loop（Reason + Act）思考 → 行動 → 觀察 → 再思考；Agent 的核心支柱：MemoryPlanningToolsAction
  • Context Engineering 與 Environment：借用 Simon's Scissors 的比喻——Agent 的表現由模型本身的能力，加上輸入的脈絡品質共同決定
  • 實現人智協作：與 Agent 共同成長：Agent 在觀察你的工作流程並結構化為可重用能力（SKILL）；你也在透過知識注入塑造它（Personal Wiki）——最終每個人的 Agent 都是獨一無二的。目前模型能力仍在持續突破，所需的系統架構輔助與人機協作方式也需要隨之調整——這不是一次性設計，而是持續演化的關係
  • 案例分享｜OpenClaw：「Agent 的 ChatGPT 時刻」：奧地利開發者一個週末獨立完成，上線一個月 GitHub 破 6 萬星。三大特性：持久記憶、主動執行、即時回報——Agent 從工具真正變成協作夥伴的臨界點
當 Agent 能力越來越強、可以直接操作你的電腦與檔案，你願意給它多大的權限？你會怎麼設定它的行為邊界？
參考資料
  • LLM Powered Autonomous Agents — Lilian Weng
  • 什麼是 Context Engineering 上下文工程？
  • 12-Factor Agents: Patterns of reliable LLM applications — Dex Horthy, HumanLayer
CH.03

如何設計 Agent 產品

20 min
  • 服務典範轉移：從操作流程設計 → 意圖系統設計（Intent-System Mapping、Outcome-Oriented Design）設計師的核心工作從「設計使用者操作步驟」轉向「定義 Agent 的行為邊界、執行準則、任務環境」，從介面設計師擴展為系統編排者
  • 個人化定義與期待的轉變：傳統個人化是「給你看什麼」——推薦錯了你滑過去就好；Agent 時代的個人化是「幫你做什麼、怎麼做」，Agent 需求了解用戶的決策邏輯與行為偏好——行動錯了是真實後果。個人脈絡的承載需要同時處理三個維度：過去（你是誰、你有什麼）、現在（你在哪、你需要什麼）、未來（你想去哪、你在意什麼），與傳統產品相比，不再是平台單方面的蒐集、預測，而是與用戶共同建構。個人化的天花板由信任深度決定：用戶願意授權 Agent 代替自己做多少事、是否提供足夠厚度的脈絡，取決於信任的累積
  • 行為邊界治理：事前規範、事中可觀測、事後錯誤邊界「AI 像實習生」——遵循漸進授權（Progressive Trust）原則，不給最高權限，不讓它在沒有 review 機制下執行關鍵動作。解釋性、可觀測性、HITL 是幫助 Agent 與用戶建立信任關係的核心議題
  • 機率系統的本質：不確定性不是缺陷，是設計條件傳統系統相同輸入產生相同輸出；AI 服務的輸出具有分佈性——設計目標從「驗證正確性」轉向「管理分佈與風險」
黃仁勳預測：未來平台級 Agent 將侵蝕大量垂直 APP 與系統服務的市場空間。所以我們還要做垂直產品嗎？未來的「產品」又是什麼？
參考資料
  • How to Embrace the Great UX Paradigm Shift to Agentic Experience Design — Salesforce
  • 12-Factor Agents: Patterns of reliable LLM applications — Dex Horthy, HumanLayer
CH.04

職能角色價值的重塑

15 min

技能被拉平後，對數位工作者帶來的影響：

  • 多種角色走向「Builder」：你是 workflow 的架構師，不只是節點Builder = 產品思維 + 設計能力 + 技術能力 + 業務能力；真正的差異不在於「會不會用 AI」，而在於能否設計整個 AI Workflow
  • 差異化來自獨特的認知交集：生成式 AI 降低了進入專業的門檻，部分可標準化的技能將被拉平；但真正的垂直專業深度與個人經驗洞察反而會變得更有價值——不可替代性來自個人獨特的跨域認知組合：觀點、品味、判斷力
  • How → What 的轉移：判斷「值得做什麼」比「怎麼做」更稀缺當 Agent 承擔大量執行任務，能清楚定義問題、設定目標、判斷產出品質的整合型判斷力（Product Sense）成為核心競爭力
參考資料
  • The design process is dead. Here's what's replacing it. | Jenny Wen (head of design at Claude)
  • Agent Engineering: A New Discipline — LangChain
  • AI 時代的產品判斷力：Product Sense 全指南 — Part 1：解構產品品味
實作 Part 動手體驗

Lab 1　打造自己的 UX 設計 Agent

30 min
目標：親身體驗 Agent 的個人化本質——感受到「這個 Agent 正在成為我的延伸」，而不只是觀察 AI 表演。
  • 小組探討自我認知中的「UX 設計流程」：你平常怎麼做設計？有哪些個人的經驗法則？
  • 盤點 Agent 需要什麼才能代替你工作：需要引入哪些參考資料與知識？需要哪些 Skills 與 Tools？
  • 根據盤點結果，構思並初步搭建屬於自己的 UX 設計 Agent

Lab 2　用 UX 設計 Agent 執行真實任務

30 min
目標：帶著 Lab 1 打造的 Agent，執行一個真實 UX 設計任務，體驗人機協作的完整流程。
  • 實際讓 Agent 執行一項 UX 設計任務，觀察它的執行過程與決策方式
  • 討論：Agent 擅長什麼？在哪些環節它做得比你好？在哪些環節它需要你介入？
  • 反思：在這個協作系統裡，人類設計者的角色定位與不可替代的價值是什麼？
